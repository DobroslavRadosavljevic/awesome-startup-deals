# Contributing to Awesome Startup Deals

Thank you for your interest in contributing to Awesome Startup Deals! This document provides guidelines and instructions for contributing.

## How to Contribute

### Reporting Issues

If you find a startup deal that's no longer active, has incorrect information, or has issues, please open an issue. Include:

- The name of the startup deal
- Description of the issue
- Link to the deal (if applicable)
- Any relevant details

### Adding New Deals

1. Fork the repository
2. Create your feature branch (`git checkout -b feature/amazing-startup-deal`)
3. Add your startup deal following the format below
4. Commit your changes (`git commit -m 'Add some amazing startup deal'`)
5. Push to the branch (`git push origin feature/amazing-startup-deal`)
6. Open a Pull Request

## Guidelines

### General Rules

- Only add startup deals that are **currently active**
- Ensure the deal has **clear terms and eligibility requirements**
- Add a brief description and eligibility criteria
- Include relevant technology badges
- Keep entries **alphabetical within each section**
- Check your spelling and grammar
- Verify that links are working and correct

### Entry Format

Each entry should follow the table format used in the README:

```markdown
| ![Provider](badge-url) **Program Name** | Offer details | Eligibility requirements | [Apply](link) |
```

**Example:**

```markdown
| ![AWS](https://img.shields.io/badge/AWS-232F3E?logo=amazonwebservices&logoColor=white) **AWS Activate** | Up to $100,000 in credits | Early-stage startups, affiliated with accelerator/VC | [Apply](https://aws.amazon.com/activate/) |
```

### Badge Guidelines

- Use appropriate technology badges (AWS, Google Cloud, Microsoft, GitHub, etc.)
- Badge format: `![Label](https://img.shields.io/badge/Label-Color?logo=logo-name&logoColor=white)`
- Use consistent badge styling throughout
- Find logo names at [Simple Icons](https://simpleicons.org/)

### Categories

Deals should be placed in the most appropriate category:

| Category | Description |
|----------|-------------|
| **Cloud Credits** | Cloud platform credits and free tiers (AWS, GCP, Azure, etc.) |
| **AI & Machine Learning** | AI platforms, LLM APIs, ML tools |
| **Development Tools** | IDEs, databases, CI/CD, monitoring |
| **Hosting & Infrastructure** | Hosting, CDN, deployment platforms |
| **SaaS & Productivity** | Project management, documentation, team tools |
| **Marketing & Analytics** | Analytics, email marketing, CRM, growth tools |
| **Communication & Collaboration** | Chat, video, email services |
| **Design Tools** | Design, prototyping, whiteboarding tools |
| **Finance & Payments** | Payment processing, banking, billing |
| **Legal & Compliance** | Incorporation, compliance, cap table management |
| **Business Services** | HR, e-signatures, general business tools |

If a deal fits multiple categories, choose the most relevant one.

### Quality Standards

- **Accuracy**: Verify all information before submitting
- **Relevance**: Only include deals specifically for startups/entrepreneurs
- **Completeness**: Include all relevant details (credits amount, eligibility, etc.)
- **Clarity**: Write clear, concise descriptions
- **Links**: Ensure all links point to official program pages

### What Makes a Good Entry

A good entry includes:

1. **Clear offer value** - Specific amounts (e.g., "$100,000 in credits") rather than vague terms
2. **Eligibility criteria** - Who qualifies (funding stage, company age, etc.)
3. **Direct link** - Link to the actual startup program page, not a generic homepage
4. **Accurate badge** - Matching the provider's brand

### What to Avoid

- Expired or discontinued programs
- Referral-only deals without public application
- Programs with unclear or hidden terms
- Duplicate entries
- Generic free tiers that aren't startup-specific

## Code of Conduct

- Be respectful and considerate
- Focus on constructive feedback
- Help maintain the quality of the list

## Review Process

- All pull requests will be reviewed
- Maintainers may request changes or ask questions
- Once approved, your contribution will be merged

## Questions?

If you have questions about contributing, feel free to open a discussion or reach out.

Thank you for helping make this list better!
